# 模型层 + 编排层 + 存储层上下文管理改造计划

## Summary

* 目标：给出一套可直接执行的明确设计，完整替换当前“单条消息热记忆 + 编排层固定读取 10 条 + provider 内部特判 system + 模型层单点裁剪”的实现。

* 约束：

  * 以当前仓库现有类型为主参考，优先复用 `ModelMessage`，避免平行造轮子。

  * 数据结构必须明确到字段和类型，非必要不使用 `Any`。

  * 函数接口必须明确到函数名、输入类型、输出类型。

  * 最终接口完全替换，不保留旧接口兼容层。

  * 关键语义不得塞进 `metadata`，仅允许调试性、非决策性信息留在 `metadata`。

* 实施边界：

  * 本轮只完成 Hot 路径、预算握手、system 归一化、工具消息原子块、QFA 闭环和 Warm/Cold/Profile 的正式空接口。

  * 不在本轮实现真实向量库、真实摘要任务、真实用户画像提取，但接口位置必须一次设计到位。

## Current State Analysis

### 当前实现

* `src/qfaos/runtime/context_facade.py`

  * `DefaultQFASessionContext._build_messages()` 直接从 `RuntimeContext.memory["dialogue_history"]` 拼接历史。

  * `add_memory()` 按单条消息调用 `append_hot_memory()`。

* `src/orchestration_engine/context/state_context_manager.py`

  * `initialize_context()` 固定 `read_hot_memory(limit=10)`。

  * `RuntimeContext.memory` 是 `dict[str, Any]`，缺少类型边界。

* `src/storage_memory/contracts/protocols.py`

  * 协议只支持单条 `HotMemoryItem` 的追加/读取。

* `src/model_provider/providers/litellm_adapter.py`

  * `trim_messages()` 已保护 tool 链不能拆，但输入仍是扁平消息列表。

* `src/model_provider/providers/minimax.py`

  * system 合并和物理裁剪放在 provider 内。

### 与目标方案的差距

* 当前没有块级上下文模型，工具消息原子性只在模型层局部成立。

* 当前没有预算握手，编排层和存储层无法控制“多读后再裁”。

* 当前没有独立的 system 片段模型，profile/snippets 未来无处接入。

* 当前 `RuntimeContext.memory`、存储协议、QFA 会话接口类型都过于松散。

* 当前旧接口命名已经把实现锁死在“hot memory message list”语义上，无法继续扩展。

## Target Design

### 1. 基础类型

新增文件：

* `src/domain/context.py`

类型定义：

```python
from __future__ import annotations

from dataclasses import dataclass, field
from typing import Literal, TypeAlias

from src.domain.models import ModelMessage

JSONScalar: TypeAlias = str | int | float | bool | None
JSONValue: TypeAlias = JSONScalar | tuple["JSONValue", ...] | dict[str, "JSONValue"]

ContextBlockKind: TypeAlias = Literal[
    "user_turn",
    "assistant_answer",
    "tool_interaction",
]

SystemPromptPartSource: TypeAlias = Literal[
    "base_prompt",
    "profile_patch",
    "memory_snippet",
]


@dataclass(frozen=True)
class ContextBudget:
    max_input_tokens: int
    reserved_output_tokens: int
    trim_ratio: float


@dataclass(frozen=True)
class SystemPromptPart:
    source: SystemPromptPartSource
    content: str


@dataclass(frozen=True)
class ContextBlock:
    block_id: str
    kind: ContextBlockKind
    messages: tuple[ModelMessage, ...]
    token_count: int


@dataclass(frozen=True)
class ContextLoadRequest:
    logic_id: str
    session_id: str
    budget: ContextBudget
    include_profile_patch: bool
    include_memory_snippets: bool
    history_block_limit: int


@dataclass(frozen=True)
class ContextLoadResult:
    system_parts: tuple[SystemPromptPart, ...]
    history_blocks: tuple[ContextBlock, ...]


@dataclass(frozen=True)
class RuntimeMemorySnapshot:
    system_parts: tuple[SystemPromptPart, ...] = ()
    history_blocks: tuple[ContextBlock, ...] = ()
```

设计说明：

* `ContextBlock.messages` 继续复用现有 `ModelMessage`。

* `token_count` 是正式字段，不放 `metadata`。

* `RuntimeMemorySnapshot` 替换当前 `dict[str, Any]` 风格的 `memory`。

### 4. 热记忆持久化模型

修改文件：

* `src/domain/memory.py`

* `src/storage_memory/internal/codecs.py`

最终定义：

```python
from dataclasses import dataclass

from src.domain.context import ContextBlockKind
from src.domain.models import ToolInvocation


@dataclass(frozen=True)
class HotMemoryItem:
    role: str
    content: str | None
    tool_calls: tuple[ToolInvocation, ...] = ()
    tool_call_id: str | None = None
    name: str | None = None
    structured_output: dict[str, object] | None = None


@dataclass(frozen=True)
class HotMemoryBlock:
    block_id: str
    kind: ContextBlockKind
    token_count: int
    messages: tuple[HotMemoryItem, ...]
```

说明：

* `HotMemoryItem` 只负责块内单条消息的可持久化表达。

* `HotMemoryBlock` 对应存储边界。

* 这里保留 `dict[str, object] | None` 的唯一原因是 `structured_output` 当前承载工具 JSON 结果；执行时需优先将其收紧到 `JSONValue` 兼容定义。

codec 必须提供的函数：

```python
def dump_hot_memory_block(block: HotMemoryBlock) -> dict[str, object]: ...
def load_hot_memory_block(payload: Mapping[str, object]) -> HotMemoryBlock: ...
def context_block_to_hot_memory_block(block: ContextBlock) -> HotMemoryBlock: ...
def hot_memory_block_to_context_block(block: HotMemoryBlock) -> ContextBlock: ...
```

## Assumptions & Decisions

* 最终接口完全替换，不保留旧接口兼容层。

* `ModelMessage` 继续作为唯一消息原语。

* `ContextBlock` 是唯一可裁剪、可持久化、可回放的上下文原子单位。

* 模型层继续承担最终物理裁剪职责；编排层和存储层承担预算内读取职责。

* 本轮不实现真实 Warm/Cold/Profile 数据源，只提供正式接口。

## 风险提示

* 若先改模型层不改存储协议，工具链原子性仍只在局部成立。

* 若继续保留旧接口，后续调用点会持续分叉，无法完成真正替换。

* 若继续把关键结构塞进 `metadata`，后续 token 预算、块裁剪、system 注入都会变得不可测试。
