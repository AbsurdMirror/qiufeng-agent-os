## T7 阶段详细文档

本文件用于拆解并落实 `T7`：重写 `qfaos` SDK 层（替代原 `app` 目录），以 Pydantic v2 的规范格式对外暴露框架接口。

## T7.1：QFAEnum 和 QFAConfig

### 1）QFAEnum / QFAConfig 设计（接口与定义）

#### QFAEnum 设计清单

| 模块 | 枚举 | 成员 | 说明 |
| :-- | :-- | :-- | :-- |
| Channel | `QFAEnum.Channel` | `Feishu` | 渠道类型枚举（首期仅飞书）。 |
| Channel | `QFAEnum.Feishu.Mode` | `long_connection` / `webhook` | 飞书运行模式枚举。 |
| Event | `QFAEnum.Event` | `TextMessage` | 统一文本消息事件类型（SDK 入口事件类型之一）。 |
| Model | `QFAEnum.Model` | `MiniMax` | 模型类型枚举（首期仅 MiniMax）。 |
| Memory | `QFAEnum.Memory.Backend` | `in_memory` / `redis` | 记忆后端类型枚举。 |
| Primitive | `QFAEnum.Primitive.Policy` | `Allow/Deny/AskTicket` | 安全原语策略决策（AskTicket 对应灰名单/需审批）。 |

#### QFAConfig 设计清单（Pydantic v2）

| 模块 | 模型 | 字段 | 类型 | 必填 | 默认值 | 说明 |
| :-- | :-- | :-- | :-- | :-- | :-- | :-- |
| Channel | `QFAConfig.Channel.Feishu` | `app_id` | `str` | 是 | - | 飞书 App ID。 |
| Channel | `QFAConfig.Channel.Feishu` | `app_secret` | `str` | 是 | - | 飞书 App Secret。 |
| Channel | `QFAConfig.Channel.Feishu` | `mode` | `QFAEnum.Feishu.Mode` | 是 | - | long_connection / webhook。 |
| Channel | `QFAConfig.Channel.Feishu` | `verify_token` | `str \| None` | 否 | `None` | Webhook/事件调度校验 token；长连接允许为空字符串。 |
| Channel | `QFAConfig.Channel.Feishu` | `encrypt_key` | `str \| None` | 否 | `None` | Webhook/事件调度加密 key；长连接允许为空字符串。 |
| Model | `QFAConfig.Model.MiniMax` | `model_name` | `str` | 是 | - | 例如 `MiniMax-Text-01`。 |
| Model | `QFAConfig.Model.MiniMax` | `api_key` | `str` | 是 | - | MiniMax API Key。 |
| Model | `QFAConfig.Model.MiniMax` | `base_url` | `str` | 否 | `https://api.minimax.chat/v1` | MiniMax Base URL。 |
| Memory | `QFAConfig.Memory` | `backend` | `QFAEnum.Memory.Backend` | 是 | - | `in_memory` 或 `redis`。 |
| Memory | `QFAConfig.Memory` | `redis_url` | `str \| None` | 否 | `None` | 当 backend=redis 时使用；默认回落到 `REDIS_URL` 或 `redis://localhost:6379`。 |
| Observability | `QFAConfig.Observability.Log` | `jsonl_log_dir` | `str` | 否 | `logs` | JSONLStorageEngine 的 `log_dir`。 |
| Observability | `QFAConfig.Observability.Log` | `jsonl_max_bytes` | `int` | 否 | `10485760` | JSONLStorageEngine 的 `max_bytes`（默认 10MB）。 |
| Observability | `QFAConfig.Observability.Log` | `jsonl_backup_count` | `int` | 否 | `5` | JSONLStorageEngine 的 `backup_count`。 |

#### 设计约束（必须满足）

- `QFAConfig` 全部模型使用 Pydantic v2（`BaseModel` + `Field` + `Annotated`）。
- `QFAConfig` 所有字段具备明确的类型、默认值和 `Field(description=...)`。
- `QFAEnum` 的枚举值与现有底层实现保持值一致或可一对一映射，避免在适配层出现大量分支。
- `QFAConfig.Channel.Feishu` 覆盖现有飞书长连接实际运行所需字段（`verify_token/encrypt_key` 即便允许为空也必须存在字段）。
- `QFAConfig.Memory` 覆盖现有 Redis 工厂 `REDIS_URL` 行为（当 `redis_url=None` 时使用环境变量或默认值）。
- `QFAConfig.Observability.Log` 覆盖现有 JSONLStorageEngine 的关键参数（log_dir/max_bytes/backup_count）。

## T7.2：QFAOS（SDK 主入口）与 Channel 注册（支持 example.py#L7-16）

本阶段的目标是：让用户能够以 SDK 方式完成“创建 QFAOS + 注册飞书 channel”的最小闭环。

> 本阶段只覆盖 `example.py#L7-16` 所需能力：`QFAOS()` 与 `register_channel(...)`。本阶段结束后，对外写法以本节“用户示例（目标形态）”为准。

### 1）对外接口设计（用户视角）

#### 用户示例（目标形态）

```python
from qfaos import QFAOS, QFAConfig, QFAEnum

agent = QFAOS()

feishu_cfg = QFAConfig.Channel.Feishu(
    app_id="123456",
    app_secret="123456",
    mode=QFAEnum.Feishu.Mode.long_connection,
)
agent.register_channel(QFAEnum.Channel.Feishu, feishu_cfg)
```

#### QFAOS 对外 API（最小集合）

- `QFAOS.__init__()`
  - 行为：创建 SDK 主对象，初始化内部注册表（channels/models/tools/memory/observability 等）。本阶段只要求 channels。
  - 约束：不触发任何网络连接，不加载任何第三方 SDK。
- `QFAOS.register_channel(channel, config) -> None`
  - 行为：注册一个渠道配置到 SDK 内部 registry。
  - 约束：
    - 仅做校验与登记，不启动运行时。
    - 同一种 channel 重复注册时，覆盖旧配置。
  - 入参约束：
    - `channel` 必须为 `QFAEnum.Channel.Feishu`。
    - `config` 必须为 `QFAConfig.Channel.Feishu`。

#### 渠道注册语义（必须明确）

- 注册结果必须能被后续运行阶段读取并使用（T7.3+）。
- `mode=webhook` 时必须校验 `verify_token/encrypt_key`，不满足直接抛错。
- `mode=long_connection` 时允许 `verify_token/encrypt_key` 为 `None`。

## T7.4：QFAOS 的安全原语与工具注册（支持 example.py#L25-61）

本阶段的目标是：让用户能够以 SDK 方式完成“注册安全原语（带策略）+ 注册工具（基于 Pydantic v2 的函数签名与 Annotated 元数据）”的最小闭环，并能被后续运行阶段读取与使用。

> 本阶段只覆盖 `example.py#L25-61` 所需能力：`register_security_primitive(...)`、`custom_pytool`、`register_tool(...)`。本阶段不启动运行时，不初始化 SkillHub/OrchestrationEngine，不产生网络请求。

### 1）对外接口设计（用户视角）

#### 用户示例（目标形态）

```python
from typing import Annotated
from pydantic import Field

from qfaos import QFAOS, QFAEnum

agent = QFAOS()

def send_email_action(to: str, subject: str, body: str) -> str:
    return f"sent to={to} subject={subject}"

def send_email_policy(to: str, _subject: str, _body: str) -> QFAEnum.Primitive.Policy:
    if to == "admin@example.com":
        return QFAEnum.Primitive.Policy.Allow
    if to.endswith("@example.com"):
        return QFAEnum.Primitive.Policy.AskTicket
    return QFAEnum.Primitive.Policy.Deny

agent.register_security_primitive("send_email", send_email_action, send_email_policy)

@agent.custom_pytool
def calculate(
    a: Annotated[int, Field(description="第一个整数")],
    b: Annotated[int, Field(description="第二个整数")],
) -> int:
    return a + b

@agent.custom_pytool
def send_email(
    to: Annotated[str, Field(description="收件人邮箱地址")],
    subject: Annotated[str, Field(description="邮件主题")],
    body: Annotated[str, Field(description="邮件正文")],
) -> str:
    return agent.primitives.send_email(to, subject, body)

agent.register_tool("calculate", calculate)
agent.register_tool("send_email", send_email)
```

#### QFAOS 对外 API（最小集合）

- `QFAOS.register_security_primitive(primitive_id, action, policy) -> None`
  - 行为：注册一个“安全原语”，并基于 `policy` 的决策对 `action` 进行安全编织，形成可执行原语。
  - 约束：
    - 注册只发生在 SDK 内部注册表，不触发运行时启动。
    - 同名 `primitive_id` 重复注册时，覆盖旧定义。
  - 入参约束：
    - `primitive_id` 必须为非空字符串。
    - `action` 必须为可调用对象。
    - `policy` 必须为可调用对象，且返回类型必须为 `QFAEnum.Primitive.Policy`。
- `QFAOS.primitives`
  - 行为：提供一个“原语命名空间对象”，使用户能够通过 `agent.primitives.<primitive_id>(...)` 调用已注册原语。
  - 约束：
    - `agent.primitives.<primitive_id>` 必须存在于注册表，否则抛错。
    - 原语执行时：
      - `Allow`：直接执行 `action` 并返回其结果。
      - `Deny`：抛出“安全拒绝”异常（用于上层捕获并形成可观测的拒绝结果）。
      - `AskTicket`：抛出“需要用户审批”异常，并携带 `ticket_id`（用于上层提示用户发起审批/重试）。
- `QFAOS.custom_pytool(func) -> func`
  - 行为：将一个 Python 函数标记为“SDK 工具函数”，并基于函数签名与 `Annotated[..., Field(...)]` 元数据生成工具输入 schema。
  - 约束：
    - 必须保留原函数可调用性。
    - 必须使用 Pydantic v2 的函数签名校验（`@validate_call`）确保运行时参数校验一致。
- `QFAOS.register_tool(tool_id, func) -> None`
  - 行为：注册一个工具函数，并在 SDK 内部生成一个可被 SkillHub 注入的 PyTool 对象。
  - 约束：
    - 注册只发生在 SDK 内部注册表，不触发运行时启动。
    - 同名 `tool_id` 重复注册时，覆盖旧定义。
  - 入参约束：
    - `tool_id` 必须为非空字符串。
    - `func` 必须为已被 `custom_pytool` 标记的函数；否则抛错。

#### 错误与返回语义（必须明确）

- `register_security_primitive(...)`：
  - `primitive_id` 为空：抛 `QFAInvalidConfigError`。
  - `policy` 返回非 `QFAEnum.Primitive.Policy`：抛 `QFAInvalidConfigError`。
- `register_tool(...)`：
  - `tool_id` 为空：抛 `QFAInvalidConfigError`。
  - `func` 未被 `custom_pytool` 标记：抛 `QFAInvalidConfigError`。

## T7.7：QFAOS 的 custom_execute 与上下文门面骨架（支持 example.py#L75-164 核心逻辑）

本阶段的目标是：让用户能够以 SDK 方式定义一个“自定义编排执行函数”（`custom_execute`）。为了让用户代码与 P0 底层的复杂机制解耦，本阶段将设计并实现 SDK 层的运行时门面对象（`QFAEvent` 与 `QFAExecutionContext`），并提供 `CustomExecuteOrchestrator` 以适配 P0 编排引擎的 `BaseOrchestrator` 规范。

> ⚠️ 注意：本阶段**不包含** `run()` 的设计与实现。`run()` 涉及六大层的装配与网络连接启动，将推迟到 T7.8 阶段。本阶段只完成“执行函数注册”与“门面适配层”的代码骨架。

### 1）对外接口设计（用户视角）

#### 用户示例（目标形态）

```python
from qfaos import QFAEnum

@my_agent.custom_execute
async def execute(event, ctx) -> None:
    if event.channel != QFAEnum.Channel.Feishu or event.type != QFAEnum.Event.TextMessage:
        return

    session_ctx = ctx.get_session_ctx(event.session_id)
    model_cfg = my_agent.models.get(QFAEnum.Model.MiniMax)
    MAX_ROUND = 5

    # 1. 处理 Ticket 恢复逻辑
    if session_ctx.state.get("ticket_asked", False):
        response = await session_ctx.call_pytool(
            session_ctx.state["tool_call"],
            ticket_id=session_ctx.state["ticket"],
        )
        assert not response.is_ask_ticket
        
        prompt = f"""
        你是一个智能体，你的任务是根据用户的问题，调用工具来回答用户的问题。
        请根据工具调用的结果继续回答用户的问题。
        你的记忆：{await session_ctx.get_memory()}
        工具调用结果：{response.output}
        """
        await session_ctx.add_memory(response.output)
        remaining_rounds = session_ctx.state["remaining_rounds"]
    else:
        prompt = f"""
        你是一个智能体，你的任务是根据用户的问题，调用工具来回答用户的问题。
        请根据用户的问题，调用工具来回答用户的问题。
        你的记忆：{await session_ctx.get_memory()}
        用户的问题：{event.payload}
        """
        await session_ctx.add_memory(event.payload)
        remaining_rounds = MAX_ROUND
    
    # 2. 进入 ReAct 循环
    for i in range(remaining_rounds):
        model_output = await session_ctx.model_ask(
            model_cfg,
            prompt,
            tools_mode="all",
        )

        if model_output.is_pytool_call:
            # 模型决定调用工具
            response = await session_ctx.call_pytool(model_output.tool_call)
            
            if response.is_ask_ticket:
                # 触发安全拦截，询问用户许可
                resp = f"""
                申请执行工具：
                工具名称：{response.tool_name}
                工具描述：{response.tool_desc}
                工具参数：{response.tool_args}
                请确认是否执行执行该工具。yes or no
                """
                await session_ctx.send_message(QFAEnum.Channel.Feishu, resp)
                session_ctx.record("qfaos.ticket_requested", {"tool_name": response.tool_name}, level="info")
                
                # 保存上下文以便下一轮恢复
                session_ctx.state = {
                    "ticket_asked": True,
                    "ticket": response.ticket,
                    "tool_call": response.tool_call,
                    "remaining_rounds": MAX_ROUND - i - 1,
                }
                break
            else:
                # 工具执行成功，将结果喂给模型
                prompt = f"""
                你是一个智能体，你的任务是根据用户的问题，调用工具来回答用户的问题。
                请根据工具调用的结果继续回答用户的问题。
                你的记忆：{await session_ctx.get_memory()}
                工具调用结果：{response.output}
                """
                await session_ctx.add_memory(response.output)
                
        elif model_output.is_answer:
            # 模型给出最终回答
            response_text = model_output.response
            await session_ctx.add_memory(response_text)
            await session_ctx.send_message(QFAEnum.Channel.Feishu, response_text)
            
            # 清理上下文
            session_ctx.state = {"ticket_asked": False}
            break
    else:
        # 达到最大轮次
        resp = f"agent工具调用次数达到上限{MAX_ROUND}"
        await session_ctx.send_message(QFAEnum.Channel.Feishu, resp)
        session_ctx.record("qfaos.max_round_reached", {"max_round": MAX_ROUND}, level="warn")

```

#### QFAOS 对外 API（最小集合）

- `QFAOS.custom_execute(func) -> func`
  - 行为：注册用户定义的异步编排入口函数。
  - 约束：
    - 必须使用 Pydantic v2 校验，`func` 必须为异步函数（`async def`）。
    - 注册阶段只做校验与保存，挂载到 SDK 内部的 `_execute_handler`。
    - 重复注册时覆盖旧定义。

#### SDK 注入的上下文与结构化对象设计（必须严格对齐 P0 OE 语义）

**1. `QFAEvent` (事件数据类)**
- 封装 P0 `UniversalEvent`。
- 暴露：`channel`, `type`, `session_id`, `payload`。

**2. `QFAModelOutput` (模型输出数据类)**
- 封装底层的模型响应解析结果。
- 暴露属性：
  - `is_pytool_call` (bool): 是否为工具调用意图。
  - `tool_call` (dict | None): 具体的工具调用载荷（若 `is_pytool_call` 为 True），其结构必须等价于 `CapabilityRequest`：包含 `capability_id/payload/metadata` 三个键。
  - `is_answer` (bool): 是否为直接文本回复。
  - `response` (str | None): 直接回复的文本内容。

**3. `QFAToolResult` (工具执行结果数据类)**
- 封装底层 `CapabilityResult`，并将安全原语的 AskTicket 结果转化为业务可处理的属性。
- 暴露属性：
  - `is_ask_ticket` (bool): 是否触发了安全拦截需要用户审批。
  - `ticket` (str | None): 审批票据 ID。
  - `tool_name` (str): 工具名称。
  - `tool_desc` (str): 工具描述。
  - `tool_args` (dict): 调用的原始参数。
  - `output` (dict): 工具执行的原始输出结果，类型与 `CapabilityResult.output` 完全一致。

**4. `QFAExecutionContext` (全局上下文门面类)**
- 针对单次事件处理周期的全局上下文管理器。
- 暴露方法：
  - `get_session_ctx(session_id: str) -> QFASessionContext`: 获取指定会话的上下文实例。本阶段实现中 `ctx` 内部只维护一个默认 session，并忽略 `session_id` 的差异；但对外必须保留该参数以锁定未来的多 session 扩展点。
  - `get_all_tools() -> list[dict]`: 遍历 `CapabilityHub` 返回当前可用工具的 Schema。

**5. `QFASessionContext` (会话上下文门面类)**
- 绑定特定 `session_id` 的操作句柄，屏蔽了 P0 的 `RuntimeContext`、`CapabilityHub`、`StorageMemory` 和 `FeishuSender` 细节。
- 属性：
  - `state` (dict): 对应底层的持久化状态字典（即 P0 `RuntimeContext.state`），用户可直接对其进行读写，执行结束后由框架自动落盘。
- 方法（必须全部为 `async` 或明确说明）：
  - `get_memory() -> list[dict]`: 获取当前会话的历史记忆（从底层 `RuntimeContext.memory` 中获取）。
  - `add_memory(content: str) -> None`: 委托给 `StorageMemory` 立即追加热记忆，内部可根据调用时机简单判断 role（如暂定统一作为 system/user 记录，或由框架层提供更细粒度的包装）。
  - `model_ask(model: QFAConfig.ModelConfigUnion, prompt: str, tools_mode: "none" | "all" | "custom" = "all", tools: tuple[CapabilityDescription, ...] | None = None) -> QFAModelOutput`:
    - 组装 `CapabilityRequest(capability_id="model.chat.default")` 并调用 `CapabilityHub.invoke`；
    - 必须以 `messages: tuple[ModelMessage, ...]` 方式传入模型输入；
    - 必须传入 `model_name`（从 `model.model_name` 读取，用于模型层路由选择）；
    - `tools_mode` 行为：
      - `"none"`：不传 tools；
      - `"all"`：传入当前 `CapabilityHub` 内所有 `domain="tool"` 的能力描述；
      - `"custom"`：传入参数 `tools` 指定的能力描述元组；
    - 必须从模型能力的 `CapabilityResult.output.tool_calls` 与 `CapabilityResult.output.content` 归一化生成 `QFAModelOutput`，禁止从 `raw` 字段自行解析 OpenAI/LiteLLM 格式。
  - `call_pytool(tool_call: dict, ticket_id: str | None = None) -> QFAToolResult`:
    - `tool_call` 必须是等价于 `CapabilityRequest` 的 dict（`capability_id/payload/metadata`）；
    - 调用 `CapabilityHub.invoke` 执行该 capability；
    - 当 `ticket_id` 非空时，必须同时：
      - 将 `ticket_id` 写入 `CapabilityRequest.ticket_id` 用于票据核销；
      - 在本次工具执行周期内为 qfaos 安全原语注入该票据，以便灰名单操作放行。
  - `send_message(channel: QFAEnum.Channel, text: str) -> None`: 委托给底层的渠道发送器回发消息。
  - `record(event_name: str, payload: dict | str, level: str = "info") -> None`: 调用 Observability Hub 的 `record(...)`，用于用户主动输出调试信息，默认自动带上当前 `trace_id`。
