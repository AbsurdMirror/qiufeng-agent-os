## 2. 测试目标与范围

### 2.1 测试目标
1. 验证 T5 新增能力的功能正确性与边界行为：响应原语与异步投递、安全原语、Schema 强校验与自愈、JSONL 调试引擎与 CLI 实时日志。
2. 提供“全链路手动测试 Agent（测试脚本）”，让测试人员可以在飞书侧通过一组明确指令覆盖 T1–T5 的关键能力。

### 2.2 范围内
- **GW-P0-07/08/09**：ReplyText、异步回传接口、飞书渠道投递适配（含群聊/私聊路由与长文本分片）。
- **SH-P0-01**：TicketStore、灰名单授权、ticket 核销、防重放、工具调用拦截器行为（CapabilityResult 错误码映射）。
- **MP-P0-05/06/07**：输出结构定义的强校验、代码块剥离、自愈重试、fail-fast。
- **OB-P0-04/05/06**：JSONL 写入、文件轮转、CLI tail + TraceID 过滤与轮转感知。
- **T1–T4 基础链路**（通过测试 Agent 串联验证）：飞书长连接接收文本、UniversalEvent 解析、SessionContext 身份映射/去重、CapabilityHub 调用模型/工具（可选模型）。

### 2.3 范围外
- 业务源码的修复与主链路补齐（只记录缺陷，不在测试阶段改 `src/`）。

## 3. 自动化测试项设计 (Automated Tests)

> 自动化测试以 pytest 为主，尽量使用 mock 避免真实外网依赖。

### 3.1 渠道层：响应原语与飞书异步投递 (GW-P0-07/08/09)
| 测试项 ID | 测试场景 | 前置条件 | 输入 / 触发动作 | 预期行为 / 验证点 |
| :--- | :--- | :--- | :--- | :--- |
| `GW-T5-01` | ReplyText 空内容防御 | 无 | `ReplyText(content="")` | 构造失败或抛错（不允许空回复原语）。 |
| `GW-T5-02` | 群聊/私聊路由选择 | 构造 UniversalEvent | 分别设置 `group_id` 有/无 | `receive_id_type` 分别为 `chat_id` / `open_id`，且 `receive_id` 选择正确。 |
| `GW-T5-03` | 长文本分片与 reply_to 规则 | `mock_mode=True` | content 长度 > 4000 | 分片数量正确；仅第一片携带 reply_to；返回结果为最后一片结果。 |
| `GW-T5-04` | tenant_access_token TTL 缓存 | mock httpx 返回 token+expire | 连续发送两次 | 第二次命中缓存不再请求鉴权；过期后会刷新。 |

### 3.2 工具层：安全原语 (SH-P0-01)
| 测试项 ID | 测试场景 | 前置条件 | 输入 / 触发动作 | 预期行为 / 验证点 |
| :--- | :--- | :--- | :--- | :--- |
| `SH-T5-01` | TicketStore 生成/有效/核销 | TTL 可控 | generate -> is_valid -> consume | consume 后 is_valid 为 False；过期后 is_valid 为 False。 |
| `SH-T5-02` | create_secure_action 灰名单流程 | policy 默认 REQUIRE_TICKET | 未带 ticket 调用 | 抛 SecurityApprovalRequiredError 且包含 ticket_id。 |
| `SH-T5-03` | with_security_policy 错误码映射 | 构造 tool 域 handler | handler 内触发 require/deny | REQUIRE_TICKET -> `error_code="requires_user_approval"`；DENY -> `error_code="security_policy_violation"`。 |
| `SH-T5-04` | 成功执行后 ticket 核销 | 先 require 再 approve | approved_ticket_id 重放 | 放行后 ticket 被 consume；再次复用同 ticket 应失败并生成新 ticket。 |

### 3.3 模型层：Schema 强校验与自愈 (MP-P0-05/06/07)
| 测试项 ID | 测试场景 | 前置条件 | 输入 / 触发动作 | 预期行为 / 验证点 |
| :--- | :--- | :--- | :--- | :--- |

| `MP-T5-02` | 无自愈时校验失败 | healing_func=None | 非法 JSON 或字段不匹配 | 抛 SchemaValidationError。 |
| `MP-T5-03` | 自愈重试后成功 | healing_func 返回修复串 | 首次失败，后续成功 | 返回 Schema 实例；重试次数符合 max_retries 语义。 |
