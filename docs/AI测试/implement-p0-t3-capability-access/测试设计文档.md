## 2. 测试目标与范围
T3 阶段的核心是建立了“一切皆能力 (Capability)”的统一路由体系。测试范围涵盖：
1. **Capability Hub**：能力的注册、发现、路由及调用时的元数据合并逻辑。
2. **Skill Hub (Browser PyTool)**：浏览器工具的轻量级探测、参数校验（如 URL 验证）及 Dry Run 模式。
3. **Model Capability Router**：模型客户端向 Capability 契约的适配转换（含 `messages`/`prompt` 的兼容解析）。
4. **MiniMax Provider (LiteLLM Adapter)**：特定模型的启发式路由判断（`is_minimax_request`）与优雅降级机制（缺少依赖/Key时不崩溃而是返回标准错误结构）。

### 3.1 编排层：Capability Hub 核心

| :--- | :--- | :--- | :--- | :--- |

### 3.4 模型层：MiniMax & LiteLLM 适配
